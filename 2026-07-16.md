### 1. CIMERA: Compute-in-Interconnect and Memory with Reconfigurable Precision for LLM Inference
- **arXiv**: [2607.13649](https://arxiv.org/abs/2607.13649)
- **作者**: Yue Jiet Chong, Yimin Wang, Wei Zhang, Xuanyao Fong
- **发布时间**: 2026-07-15

**摘要**:
LLM 对计算和内存提出了巨大挑战，从数据中心到边缘设备都需要高能效推理。权重精度在平衡推理准确率、吞吐量和能耗中扮演关键角色，而现代 LLM 工作负载表现出显著的异构性和容错性，有利于自适应精度执行。本文提出 CIMERA，一种可重构精度的 LLM 推理加速器，集成了互联和内存中的计算（Compute-in-Interconnect and Memory），以缓解内存墙并实现精度感知执行。与 NVIDIA H100 相比，CIMERA 在 1B 和 13B 模型上分别实现了高达 25 倍和 10 倍的能效提升。

**中文翻译**:
CIMERA 是一种面向 LLM 推理的可重配置精度加速器，通过在互联和内存中集成计算来缓解内存瓶颈。其核心思路是利用 LLM 权重精度可异构调整的特性，在不同部署场景（数据中心到边缘）下自适应选择精度，在 H100 对比下达到 10-25× 能效提升。

### 2. Kaleido: Algorithm-Hardware Co-Design for Video Diffusion Transformers by Exploiting Latent Space Correlations
- **arXiv**: [2607.13770](https://arxiv.org/abs/2607.13770)
- **作者**: Wenxuan Miao, Haosong Liu, Weiming Hu, Zihan Liu, Aiyue Chen, Jianlin Yu, Yiwu Yao, Yiming Gan, Jieru Zhao, Jingwen Leng, Minyi Guo, Yu Feng
- **发布时间**: 2026-07-15

**摘要**:
视频扩散 Transformer（vDiT）能生成高质量视频，但由于长扩散时间步和自注意力计算带来了极高的计算成本。随着扩散时间步减少，自注意力的计算成本成为主要瓶颈。现有加速方法大多继承自 LLM 的稀疏注意力技术，未能考虑视频数据独特的时空相关性。本文提出 Kaleido，一种算法-硬件协同设计方法，通过利用潜空间中的通道级时空相关性来加速 vDiT 中的所有操作。基于此洞察，我们提出一种轻量级通道级重用算法，通过重用部分结果跳过冗余计算，同时比现有方法保持更高的生成质量（>17 dB）。为了高效支持该算法，我们设计了类似脉动阵列的加速器，具有可重构处理单元和轻量级数据调度器，以缓解重用算法引入的不规则稀疏性和数据访问模式。在三种主流 vDiT 模型上的评估表明，Kaleido 相比最先进的加速器实现了高达 5.9 倍加速和 16.0 倍能效提升。

**中文翻译**:
Kaleido 针对视频扩散 Transformer（vDiT）推理慢的问题，利用视频数据在潜空间中的通道级时空相关性，设计了一种轻量级结果重用算法来跳过冗余计算。同时配套设计了类似脉动阵列的硬件加速器，包含可重构处理单元和轻量级数据分发器。在主流 vDiT 模型上相对 SOTA 加速器实现 5.9× 加速和 16× 能效提升。

### 3. Attention-Free and Lightweight Token Reduction for Efficient Vision-Language Models

- **发布时间**: 2026-07-15

**摘要**:
视觉语言模型（VLM）在多模态理解中取得了强大性能，但由于处理大量视觉 Token 带来的巨大计算开销，在资源受限的边缘设备上部署仍然具有挑战性。Token 缩减是加速 VLM 推理的一个有前景的方向，但现有方法要么依赖与现代加速框架不兼容的注意力图，要么依赖计算密集的成对相似性比较，这削弱了可扩展性并抵消了实际部署中的收益。本文提出一种无注意力（attention-free）的轻量级 Token 缩减框架，作为 VLM 的即插即用模块。它保留重要且多样化的 Token，生成紧凑的视觉表示。首先，为了实现无注意力的重要性估计，我们采用信息论视角，使用一种新颖的基于熵的准则量化 Token 信息。其次，为确保轻量级覆盖多样性，我们引入了一种变换诱导的一致性信号，相似 Token 产生相似信号，通过对该信号排序使相似 Token 相邻，并通过步长选择产生多样化的 Token 集。广泛的实验表明，该框架在激进压缩下保持了有竞争力的准确率-效率平衡。

**中文翻译**:

### 4. ExTernD: Expanded-Rank Ternary Decomposition Ternary LLM PTQ with Accuracy Approaching Any Quantization Level
- **arXiv**: [2607.13511](https://arxiv.org/abs/2607.13511)
- **作者**: Chethan Reddy G. P
- **发布时间**: 2026-07-15

**摘要**:
我们提出 ExTernD（扩展秩三元分解），一种将每个 LLM 权重矩阵分解为三元因子的后训练方法：$A \approx B \cdot \text{diag}(D) \cdot C$，其中三元因子 $B, C \in \{-1,0,+1\}$, 实值尺度向量 $D$。内秩 $k = \mu \cdot \min(m,n)$ 被有意扩展到全秩之外（$\mu > 1$），使得超过全秩的分量能够纠正前面分量的量化误差。我们证明残差随 $k$ 单调递减，并且可以降至任意 $\varepsilon > 0$ 以下：ExTernD 可以任意接近 bf16 精度，这是任何固定平面数的三元方案无法做到的。内存和计算随 $\mu$ 连续扩展，因子稀疏度随阈值 $\tau$ 连续变化，因此可以精确命中精度目标。ExTernD 在 5.2-5.5 有效 bpw 下匹配 Q4_K 的每矩阵精度，在 Gemma-4-E2B 和 Qwen3.5-4B 上达到 ~5.7 有效 bpw 的 Q4_K/Q5_K 精度区间。

**中文翻译**:
ExTernD 将 LLM 权重矩阵分解为两个三元（-1,0,+1）矩阵和一个实数尺度向量，且内秩被刻意扩展到全秩以上（$\mu > 1$），使超出全秩的分量能够纠正早期分量的量化误差。理论上可无限逼近 bf16 精度量化方案做不到。在 5.2–5.5 有效 bpw 下匹配 Q4_K 精度，为部署场景提供了灵活精度-计算权衡。

- **作者**: Niels Mündler-Sasahara, Hristo Venev, Dawn Song, Martin Vechev, Jingxuan He
- **发布时间**: 2026-07-15

**摘要**:
具有丰富静态语义的语言（如 Rust）为 AI 生成代码提供了更强的保证，但它们的严格性使生成变得更加困难。现成的编译器可以在生成后提供有用的反馈，但无法指导中间生成步骤（如自回归 LLM 解码期间）。约束解码（Constrained decoding）更早介入，在采样期间拒绝无效 Token，但需要白盒模型访问和对语义约束代价高昂的重新实现。我们引入生成式编译（Generative Compilation），第一种在生成过程中获取部分程序编译器反馈的方法。核心技术设备是 sealor：一种轻量级的、主要由语法引导的转换，将部分程序转换为完整程序，使得标准编译器可以诊断。我们构建了这种 sealor 并证明其性质（在 Lean 中机械化验证），将其扩展到实际的 Rust 部分程序检查器。在仓库级 Rust 编码任务上的评估表明，生成式编译减少了非编译输出并提高了功能正确性。

**中文翻译**:
针对 Rust 等强类型语言中 AI 生成代码编译困难的问题，提出"生成式编译"方法。核心是 sealor——一种轻量级语法引导转换工具，将部分程序转化为标准编译器能诊断的完整程序，从而在代码生成过程中即时提供编译器反馈（而非生成后检查）。在仓库级 Rust 任务上减少了编译失败并提升了代码正确性。
