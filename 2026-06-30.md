### 1. TraceLab: Characterizing Coding Agent Workloads for LLM Serving
**🔗** [https://arxiv.org/abs/2606.30560](https://arxiv.org/abs/2606.30560)

**Abstract:**
Coding agents are rapidly becoming a major application of agentic LLMs, but serving them efficiently remains challenging. Progress on this challenge requires understanding real workload patterns, yet existing serving benchmarks predominantly focus on single-turn, single-request interactions. We introduce TraceLab, a framework for characterizing, benchmarking, and synthesizing coding agent workloads. We conduct a large-scale measurement study of coding agent interactions from a production deployment, collecting over 300K user sessions and 75M tokens. From these traces, we identify key workload characteristics: (i) sessions are highly variable in request count, inter-arrival time, and token consumption; (ii) request patterns within sessions exhibit strong temporal structure and prompt-context locality; and (iii) token generation-to-prompt ratios follow a long-tailed distribution. We characterize the scheduling and allocation implications of these patterns for LLM serving systems, identifying opportunities for request-level optimizations including prefill-aware scheduling and session-aware caching.

### 2. Energy-Aware Scheduling for Serverless LLM Serving on Shared GPUs
**🔗** [https://arxiv.org/abs/2606.30391](https://arxiv.org/abs/2606.30391)

**Abstract:**
As LLM inference becomes a major cloud workload, its growing energy footprint makes cluster-wide energy optimization increasingly important. Serverless LLM serving helps platforms absorb traffic volatility but introduces scheduling challenges due to GPU memory constraints and variable request patterns. We present an energy-aware scheduling framework for serverless LLM serving that jointly optimizes request routing and GPU allocation to reduce energy consumption while meeting latency SLOs. Our scheduler leverages a lightweight performance model to predict inference latency and power consumption across heterogeneous GPU configurations, and formulates scheduling as a constrained optimization problem. We implement a prototype on a cluster of NVIDIA GPUs and evaluate with production LLM workloads. Results show that our approach reduces total energy consumption by 32-47% compared to latency-optimized schedulers while maintaining P99 latency within SLO targets.

### 3. HBM Is Not All You Need: Efficient Disaggregated LLM Serving across Memory-heterogeneous Accelerators
**🔗** [https://arxiv.org/abs/2606.29986](https://arxiv.org/abs/2606.29986)

**Abstract:**
LLM inference comprises a compute-bound prefill phase and a memory-bound decode phase, and recent systems disaggregate them onto separate hardware. Yet today's datacenter GPUs rely on costly HBM whose capacity falls short of growing LLM model sizes, forcing expensive memory offloading. We observe that the memory-bound decode phase does not require the full bandwidth of HBM; it only needs sufficient capacity to hold the KV cache. This insight motivates attention-offloaded disaggregated serving (AODS), a disaggregated LLM serving system that pairs high-compute GPUs (HBM-rich) for prefill with memory-capacity-optimized accelerators for decode. We design decode accelerators using GDDR-based GPUs, which offer 3-4x higher memory capacity per dollar than HBM. We build a prototype and show that AODS achieves 2.3x higher serving throughput and 1.8x lower cost compared to HBM-only disaggregated serving.

**中文摘要:**
LLM推理包括计算密集的预填充阶段和内存密集的解码阶段，最近的系统将两者解耦到不同的硬件上。然而，当今数据中心GPU依赖昂贵的HBM，其容量难以满足不断增长的LLM模型大小，迫使进行昂贵的内存卸载。我们观察到，内存密集的解码阶段并不需要HBM的全部带宽；它只需要足够的容量来容纳KV缓存。这一见解催生了注意力卸载解耦服务（AODS），这是一个解耦的LLM服务系统，将高计算能力GPU（富HBM）用于预填充，将内存容量优化的加速器用于解码。我们使用基于GDDR的GPU设计了解码加速器，其每美元内存容量比HBM高3-4倍。我们构建了原型，并展示了AODS相比纯HBM解耦服务，实现了2.3倍更高的服务吞吐量和1.8倍更低的成本。

**🔗** [https://arxiv.org/abs/2606.29844](https://arxiv.org/abs/2606.29844)

**Abstract:**
The quadratic computational cost of traditional attention mechanisms poses a major bottleneck to the scalability and practical deployment of large language models (LLMs), particularly in long-context scenarios. We propose MATCH (Modulating Attention via in-Context retrieval), a method that reduces attention complexity by replacing full quadratic attention with a sparse attention pattern modulated by in-context retrieval. MATCH dynamically selects a subset of tokens from the context to attend to, using a fast retrieval mechanism that identifies relevant tokens based on their semantic similarity to the current query. **Abstract:**
