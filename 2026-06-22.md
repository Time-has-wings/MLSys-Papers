### 1. Execution-State Capsules: Graph-Bound Execution-State Checkpoint and Restore for Low-Latency, Small-Batch, On-Device Physical-AI Serving

- **论文 ID**：[`2606.20537`](https://arxiv.org/abs/2606.20537)（2026-06-18）
- **作者**：Liang Su
- **分类**：cs.LG, cs.DC
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.20537) | [📑 PDF](https://arxiv.org/pdf/2606.20537)

**英文摘要（English Abstract）：**
> Mainstream LLM serving systems reuse prefix work mainly through paged or radix key-value (KV) caches. This is highly effective for high-throughput, high-concurrency serving, but it manages only one positional fragment of execution state: the KV cache. We study the opposite regime: low-latency, small-batch, on-device physical-AI serving, where interactive LLM agents, speech systems, and robot policies repeatedly branch, reset, interrupt, and re-enter under tight responsiveness budgets. We introduce execution-state capsules, a graph-bound checkpoint and restore mechanism for the complete restorable state at a committed boundary. FlashRT is a white-box, backend-facing kernel runtime whose evaluated NVIDIA CUDA backend runs captured graph plans over contiguous static buffers with no block-table indirection. Because the live state is a closed set of named buffers, a capsule can snapshot, restore, fork, or roll back the whole execution boundary, including KV, recurrent state, convolution state, MTP state, and metadata. This moves reuse from token-addressed KV fragments to graph-bound execution-state boundaries. On an RTX 5090, capsule restore is byte-exact at the stored-state level and token-identical under greedy decode. A KV-only ablation diverges, showing that recurrent state is load-bearing. GPU-resident snapshot and restore are sub-millisecond, and TTFT speedup over cold prefill grows from 3.9x at 2k tokens to 27x at 16k tokens. On Jetson AGX Thor and DGX Spark, the same correctness and structural properties hold. Capsules are not a replacement for high-throughput KV-cache serving; they define a complementary latency-first serving point for explicit execution-state reuse.

**中文摘要：**
> 【领域】端侧 LLM 推理 / 低延迟 Serving
> 【问题】主流 LLM 推理系统通过 PagedAttention 或 RadixAttention 复用 KV Cache 来加速高吞吐、高并发场景，但只管理了执行状态的一个片段（KV Cache），在端侧物理 AI 场景（低延迟、小批量、交互式 LLM Agent / 语音 / 机器人策略）中效率不足。
> 【方法】本文提出 **Execution-State Capsules（执行状态胶囊）**，一种基于计算图边界的完整执行状态检查点与恢复机制。核心系统 **FlashRT** 是一个白盒后端 Kernel 运行时，将捕获的计算图计划运行在连续静态缓冲区上（无 Block Table 间接跳转）。胶囊可对整个执行边界（包括 KV、循环状态、卷积状态、MTP 状态和元数据）进行快照、恢复、分支和回滚，将复用从「Token 寻址的 KV 片段」升级为「计算图绑定的执行状态边界」。
> 【效果】在 RTX 5090 上，胶囊恢复在存储状态级别是字节级精确的，贪心解码下 Token 一致。仅恢复 KV 的消融实验出现发散，证明循环状态是关键的。GPU 驻留快照和恢复均在亚毫秒级别，TTFT 加速比从 2K Token 时的 3.9× 提升至 16K Token 时的 27×。在 Jetson AGX Thor 和 DGX Spark 上也有相同的正确性和结构性保证。

### 2. CacheWeaver: Cache-Aware Evidence Ordering for Efficient Grounded RAG Inference

- **论文 ID**：[`2606.19667`](https://arxiv.org/abs/2606.19667)（2026-06-18）
- **作者**：Kaizhen Tan, Rong Gu, Mingyuan Li
- **分类**：cs.CL
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.19667) | [📑 PDF](https://arxiv.org/pdf/2606.19667)

**英文摘要（English Abstract）：**
> Retrieval-Augmented Generation (RAG) improves factual grounding, but it also lengthens prompts and raises prefill cost. Prefix caching in serving engines such as vLLM reduces this cost only when requests share the same token prefix. In grounded generation, however, adjacent queries may retrieve overlapping evidence in different orders, so set overlap does not become reusable prefix overlap. We present CacheWeaver, a lightweight prompt-layer method for cache-aware evidence ordering. The method keeps a prefix tree over recently served evidence sequences and uses a greedy walk to place the most reusable prefix first, while leaving the serving engine and retrieved evidence set unchanged. Across three vLLM configurations, the method lowers median time-to-first-token (TTFT) by about 20-33 percent relative to retrieval-order prefix caching, without hurting answer quality in our QA tests. The greedy policy reaches 97.5 percent of the median TTFT gain from oracle ordering, indicating that most reusable prefix locality can be recovered by a simple scheduling layer between retrieval and inference.

**中文摘要：**
> 【领域】RAG 推理 / Prefix Cache 优化
> 【问题】RAG 通过检索增强生成的事实准确性，但拉长了 Prompt 长度并增加了 Prefill 开销。vLLM 等推理引擎的 Prefix Caching 仅在请求共享相同 Token 前缀时才能降低开销，而 RAG 场景中相邻查询检索到的证据集可能内容重叠但顺序不同，无法复用前缀缓存。
> 【方法】本文提出 **CacheWeaver**，一种轻量级的 Prompt 层缓存感知证据排序方法。它维护一个最近服务过的证据序列的前缀树，通过贪心遍历将最可复用的前缀放在最前面，不修改推理引擎和检索到的证据集合本身。
> 【效果】在三种 vLLM 配置下，相比检索顺序的 Prefix Caching，中位数 TTFT 降低 **20-33%**，且不影响答案质量。贪心策略达到了 Oracle 排序中位数 TTFT 收益的 **97.5%**，说明大部分可复用的前缀局部性可以通过检索和推理之间的简单调度层恢复。

### 3. ReMP: Low-Downtime Runtime Model-Parallelism Reconfiguration for LLM Serving

- **论文 ID**：[`2606.18741`](https://arxiv.org/abs/2606.18741)（2026-06-17）
- **作者**：Haipeng Yuan, Kaining Zheng, Yongshu Bai, Yuchen Zhang, Yunquan Zhang, Baodong Wu, Xiang Gao, Daning Cheng
- **分类**：cs.DC
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.18741) | [📑 PDF](https://arxiv.org/pdf/2606.18741)

**英文摘要（English Abstract）：**
> Current large language model (LLM) inference systems universally deploy ultra-large-scale models using a combination of Tensor Parallelism (TP) and Pipeline Parallelism (PP). However, existing systems treat the model parallelism topology as a static configuration that cannot be flexibly adjusted at runtime. This rigid design creates a fundamental contradiction with the dynamically changing inference workloads in real-world scenarios. State-of-the-art systems lack online reconfiguration capabilities and can only switch configurations by restarting the service, resulting in several minutes of service interruption, KV cache loss, and prohibitive recomputation overhead. To address this problem, this paper presents ReMP, a runtime model parallelism reconfiguration framework that supports low downtime. ReMP achieves dynamic adjustment through three key techniques: (1) decoupling the model parallelism topology from runtime state to avoid full service reconstruction; (2) designing a two-dimensional KV cache migration mechanism to preserve reusable cache states after TP/PP changes; and (3) implementing end-to-end online reconfiguration. Experiments demonstrate that ReMP can complete most topology switches within 1-7 seconds on models ranging from 7B to 70B parameters, achieving speedups of tens to over a hundred times compared to the restart approach. Moreover, ReMP significantly outperforms fixed configurations under dynamic workloads, delivering superior performance in terms of TTFT, TPOT, and output throughput.

**中文摘要：**
> 【领域】LLM 推理 / 模型并行重配置
> 【问题】当前 LLM 推理系统普遍使用 TP（张量并行）和 PP（流水线并行）部署超大规模模型，但拓扑是静态配置的，无法根据动态变化的推理工作负载灵活调整。切换配置只能重启服务，导致数分钟的服务中断、KV Cache 丢失和巨大的重计算开销。
> 【方法】本文提出 **ReMP**，一种支持低停机的运行时模型并行重配置框架。三项关键技术：(1) 将模型并行拓扑与运行时状态解耦，避免完全重建服务；(2) 设计二维 KV Cache 迁移机制，在 TP/PP 变化后保留可复用缓存状态；(3) 端到端在线重配置实现。
> **【效果】** 在 7B 到 70B 参数的模型上，ReMP 在 **1-7 秒**内完成大多数拓扑切换，相比重启方法加速数十到上百倍。在动态工作负载下显著优于固定配置，在 TTFT、TPOT 和输出吞吐量方面均表现优异。

### 4. SafeSpec: Fast and Safe LLM via Dynamic Reflective Sampling

- **论文 ID**：[`2606.19755`](https://arxiv.org/abs/2606.19755)（2026-06-18）
- **作者**：Haotian Xu, Zeyang Zhang, Linbao Li, Huadi Zheng, Yu Li, Cheng Zhuo
- **分类**：cs.CR, cs.AI
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.19755) | [📑 PDF](https://arxiv.org/pdf/2606.19755)

**英文摘要（English Abstract）：**
> Speculative inference accelerates large language model (LLM) decoding but provides no inherent safety guarantees. Existing safety defenses are largely incompatible with speculative inference: they either introduce additional computation or disrupt the draft-verify mechanism, negating acceleration benefits. This reveals a fundamental incompatibility between current safety methods and speculative decoding. We propose SafeSpec, a safety-aware speculative inference framework that integrates risk estimation directly into the verification process. SafeSpec attaches a lightweight latent safety head to the target model to jointly evaluate semantic validity and safety in a single forward pass. When unsafe generations are detected, SafeSpec applies rollback and safety-guided reflective multi-sampling to recover safe continuations rather than terminating generation. We model jailbreak attacks as distributional shifts over generative trajectories, where adversarial prompts increase the probability of harmful continuations without eliminating safe ones. Under this model, SafeSpec performs risk-aware trajectory recovery within the speculative decoding process. Across multiple models and adversarial benchmarks, SafeSpec achieves a substantially improved safety-efficiency trade-off. On Qwen3-32B, SafeSpec reduces attack success rates by 15% while preserving a 2.06x inference speedup on benign workloads, demonstrating that speculative acceleration and inference-time safety can be jointly optimized.

**中文摘要：**
> 【领域】投机推理 / LLM 安全
> 【问题】投机推理（Speculative Decoding）加速了 LLM 解码，但本身不提供任何安全保障。现有安全防御与投机推理不兼容——要么引入额外计算，要么破坏 Draft-Verify 机制，抵消加速收益。
> 【方法】本文提出 **SafeSpec**，一种安全感知的投机推理框架，将风险评估直接集成到验证过程中。核心创新：在目标模型上附加一个轻量级潜在安全头（Latent Safety Head），在单次前向传播中联合评估语义有效性和安全性。检测到不安全生成时，SafeSpec 执行回滚和安全引导的反射式多采样（Reflective Multi-Sampling）恢复安全续写，而非终止生成。
> **【效果】** 在 Qwen3-32B 上，攻击成功率降低 **15%**，同时保持 **2.06×** 推理加速比，证明了投机加速和推理时安全可以联合优化。

### 5. EfficientRollout: System-Aware Self-Speculative Decoding for RL Rollouts

- **论文 ID**：[`2606.18967`](https://arxiv.org/abs/2606.18967)（2026-06-17）
- **作者**：Minseo Kim, Minjae Lee, Seunghyuk Oh, Kevin Galim, Donghoon Kim, Coleman Hooper, Harman Singh, Amir Gholami, Hyung Il Koo, Wonjun Kang
- **分类**：cs.LG
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.18967) | [📑 PDF](https://arxiv.org/pdf/2606.18967)

**英文摘要（English Abstract）：**
> Reinforcement learning (RL) has become a representative post-training paradigm for LLMs, enabling strong reasoning and agentic capabilities. However, rollout generation remains a dominant latency bottleneck because autoregressive sampling decodes responses sequentially and a small number of long-tailed generations often determine completion time. Speculative decoding (SD) offers a natural way to address this bottleneck, as it is a well-established technique for serving fixed LLMs that reduces latency by rapidly drafting tokens and accepting them through parallel verification while preserving the target-model distribution. However, its practical speedups do not directly carry over to RL rollouts: (i) the evolving target policy makes any fixed drafter increasingly mismatched with the policy's output distribution; and (ii) active batch sizes shrink throughout rollout decoding, shifting decoding from compute-bound to memory-bound regimes where parallel verification can exploit underutilized compute. Therefore, accelerating RL rollouts requires both a drafter that remains effective under long, high-temperature generations from an evolving policy and system-aware use of SD that avoids compute-bound regimes. We present EfficientRollout, a system-aware self-SD framework designed to address this gap for RL rollouts. EfficientRollout induces a quantized drafter from the target model (i.e. self-speculative decoding), keeping it coupled to the evolving policy without separate drafter pretraining or online adaptation. It further coordinates a system-aware SD toggle policy with acceptance-aware draft-length adaptation, enabling speculation only in beneficial regimes while matching the drafting budget to evolving drafter quality. EfficientRollout reduces rollout and end-to-end latency by up to 19.6% and 12.7%, respectively, over an accelerated AR rollout baseline, while preserving final model quality.

**中文摘要：**
> 【领域】RL Rollout / 投机解码
> 【问题】强化学习（RL）已成为 LLM 后训练的主流范式，但 Rollout 生成是关键的延迟瓶颈。投机解码（Speculative Decoding）在固定模型推理上效果显著，但无法直接用于 RL Rollouts：(1) 目标策略不断演化，固定 Draft 模型与策略输出分布不匹配；(2) 活跃 Batch Size 随 Rollout 缩小，解码从计算密集型转向存储密集型。
> 【方法】本文提出 **EfficientRollout**，一种面向 RL Rollout 的系统感知自投机解码框架。它从目标模型中诱导一个量化 Draft 模型（自投机解码，无需额外预训练或在线适应），并结合系统感知的 SD 开关策略和接受率感知的 Draft 长度自适应机制，仅在有益状态下启用投机。
> **【效果】** 相比加速后的 AR Rollout 基线，Rollout 延迟降低 **19.6%**，端到端延迟降低 **12.7%**，且保持最终模型质量不变。

### 6. ViCoStream: Streaming VideoLLMs Can Run Beyond 100 FPS with Stage-Wise Coordinated Inference

- **论文 ID**：[`2606.19849`](https://arxiv.org/abs/2606.19849)（2026-06-18）
- **作者**：Yang Tan, Junlong Tong, Linan Yue, Hao Wu, Pengfei Fang, Xiaoyu Shen
- **分类**：cs.CV
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.19849) | [📑 PDF](https://arxiv.org/pdf/2606.19849)

**英文摘要（English Abstract）：**
> Streaming VideoLLMs must continuously process incoming video while maintaining low query latency, making both video-ingestion throughput and query-time responsiveness critical for real-time deployment. Existing methods largely focus on accelerating individual modules, such as visual encoding, token pruning, or KV-cache compression, but provide limited insight into whether the resulting system can sustain real-time streaming performance. We formulate streaming VideoLLM inference as a coordinated pipeline spanning visual preprocessing, visual encoding, token dropping, and LLM prefilling/decoding. Building on this formulation, we propose ViCoStream (Video Coordinated Streaming), a stage-wise coordinated streaming framework that combines chunk-wise execution, CUDA-stream overlap, visual token control, bounded visual attention, and query-side retrieval to bound per-chunk computation and memory costs. We further provide a systematic study of bottleneck migration, revealing how chunk size, token retention, attention locality, and retrieval scope shape the throughput-accuracy trade-off. Experiments with Qwen2.5-VL-3B/7B-Instruct across multiple streaming benchmarks show that ViCoStream achieves 134 FPS video throughput and less than 50 ms TTFT on a single A100 GPU while maintaining accuracy close to full-history baselines.

**中文摘要：**
> 【领域】流式 VideoLLM 推理
> 【问题】流式 VideoLLM 需要持续处理输入视频同时保持低查询延迟，现有方法聚焦于加速单个模块（视觉编码、Token 剪枝、KV Cache 压缩），但缺乏对系统能否维持实时流式性能的整体判断。
> 【方法】本文将流式 VideoLLM 推理形式化为一个跨视觉预处理、视觉编码、Token 丢弃和 LLM Prefill/Decode 的协调流水线，提出 **ViCoStream**，一种阶段式协调流式推理框架。它结合 Chunk 式执行、CUDA-Stream 重叠、视觉 Token 控制、有界视觉注意力和查询侧检索，限制每 Chunk 的计算和存储成本。
> **【效果】** 在单张 A100 GPU 上，Qwen2.5-VL-3B/7B-Instruct 实现 **134 FPS** 视频吞吐量和 **<50ms** TTFT，同时准确率接近全历史基线。

### 7. UltraQuant: 4-bit KV Caching for Context-Heavy Agents

- **论文 ID**：[`2606.20474`](https://arxiv.org/abs/2606.20474)（2026-06-18）
- **作者**：Inesh Chakrabarti, David Limpus, Aditi Ghai Rana, Bowen Bao, Spandan Tiwari, Thiago Crepaldi, Ashish Sirasao
- **分类**：cs.LG, cs.AI, cs.PF
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.20474) | [📑 PDF](https://arxiv.org/pdf/2606.20474)

**英文摘要（English Abstract）：**
> Context-heavy agents place unusual pressure on the key-value (KV) cache: long prefixes are reused across many short turns, while concurrency determines whether the serving system can keep GPUs utilized. We study 4-bit KV-cache compression for this setting, using TurboQuant-style rotation and codebook quantization as a quality anchor and vLLM FP8 KV caching as the deployment anchor. We report three contributions. First, we frame 4-bit KV caching around multi-round agent workloads where task quality, cache residency, and serving throughput must be measured jointly. Second, we describe the practical design choices needed to make the 4-bit path robust, including asymmetric K/V treatment, Walsh-Hadamard rotation, QJL removal, and block-scale variants. Third, we present serving optimizations on AMD GPUs, including optimized decode-attention kernels and UltraQuant, an FP4 approximation path that uses FP8 queries, FP4 KV tensors, UE8M0 group scales, and native scaled-MFMA support on CDNA4. On a long-context, multi-turn agentic workload, UltraQuant cuts P50 time-to-first-token by 3.47x in the cache-pressured late rounds (2.3x across all rounds) and raises output throughput by 1.63x over the FP8 KV baseline.

**中文摘要：**
> 【领域】KV Cache 量化 / Agent 推理优化
> 【问题】上下文密集的 Agent 场景对 KV Cache 造成异常压力：长前缀跨多轮短交互复用，并发度决定 GPU 利用率。4-bit KV Cache 压缩在此场景下具有巨大潜力。
> 【方法】本文研究 4-bit KV Cache 压缩，提出 **UltraQuant**。关键技术：(1) 将 4-bit KV 缓存围绕多轮 Agent 工作负载进行框架化设计，联合衡量任务质量、缓存驻留和服务吞吐量；(2) 实际设计选择包括非对称 K/V 处理、Walsh-Hadamard 旋转、QJL 移除、Block-Scale 变体；(3) AMD GPU 服役优化，包括优化的 Decode-Attention Kernel 和 FP4 近似路径（FP8 Query、FP4 KV、UE8M0 组缩放、CDNA4 原生 Scaled-MFMA 支持）。
> **【效果】** 在长上下文多轮 Agent 工作负载上，缓存压力的后期轮次 P50 TTFT 降低 **3.47×**（全轮次平均 2.3×），输出吞吐量比 FP8 KV 基线提升 **1.63×**。

### 8. Closing the Calibration Gap in Semantic Caching

- **论文 ID**：[`2606.19719`](https://arxiv.org/abs/2606.19719)（2026-06-18）
- **作者**：Aditeya Baral, Radoslav Ralev, Iliya Sotirov Zhechev, Srijith Rajamohan, Jen Agarwal
- **分类**：cs.IR, cs.CL, cs.LG
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.19719) | [📑 PDF](https://arxiv.org/pdf/2606.19719)

**英文摘要（English Abstract）：**
> Semantic caching cuts LLM inference costs by serving a cached response to semantically similar queries. Standard practice evaluates these systems using PR-AUC, a metric that only measures how well scores rank and ignores whether they are usable at a fixed threshold. We show this mismatch leads to systematically poor deployment choices, as models with the highest PR-AUC are often the worst in operation. We introduce Precision-Cache Hit Ratio (P-CHR) AUC, a cache-aware metric that measures precision across cache utilization levels, and Calibration Retention Rate (CRR), which captures how much offline ranking quality survives at deployment. We decompose the operational gap between offline and deployed quality into a recoverable calibration component and an irreducible structural component fixed by the dataset's positive rate. Our experiments show that the calibration gap is governed by the training objective rather than data scale, and post-hoc calibration only partially closes it. Ultimately, model selection for semantic caching is a calibration problem, not a ranking one, and measuring it is the first step to closing the gap.

**中文摘要：**
> 【领域】语义缓存 / LLM 推理成本优化
> 【问题】语义缓存（Semantic Caching）通过为语义相似的查询返回缓存响应来降低 LLM 推理成本。现有评估使用 PR-AUC 指标，但这只衡量分数排序质量，忽略了在固定阈值下的可用性，导致最高 PR-AUC 的模型在实际部署中往往表现最差。
> 【方法】本文提出两项新指标：(1) **P-CHR AUC**（Precision-Cache Hit Ratio AUC），一种缓存感知的度量，衡量跨缓存利用水平的精度；(2) **CRR**（Calibration Retention Rate），捕捉离线排序质量在部署中有多少得以保留。将离线/部署质量差距分解为可恢复的校准组件和不可约的结构组件。
> **【发现】** 校准差距由训练目标而非数据规模决定，事后校准仅能部分弥合该差距。语义缓存的模型选择本质上是一个校准问题，而非排序问题。

### 9. Techniques for Peak Memory Reduction for LoRA Fine-tuning of LLMs on Edge Devices

- **论文 ID**：[`2606.19528`](https://arxiv.org/abs/2606.19528)（2026-06-17）
- **作者**：Hassan Dbouk, Matthias Reisser, Prathamesh Mandke, Likhita Arun Navali, Christos Louizos
- **分类**：cs.LG, cs.AI
- **链接**：[📄 arXiv](https://arxiv.org/abs/2606.19528) | [📑 PDF](https://arxiv.org/pdf/2606.19528)

**英文摘要（English Abstract）：**
> Fine-tuning of Large Language Models (LLMs) using Low-Rank Adaptation (LoRA) on an end-user's data offers personalized experiences while keeping data private, but faces severe memory constraints on consumer hardware. Peak memory during fine-tuning often exceeds device limits, especially for models with billions of parameters and long-context training data. This paper introduces a suite of complementary techniques to reduce memory footprint without sacrificing model quality: (1) base model quantization with on-the-fly dequantization, (2) memory-efficient checkpointing combining selective activation caching and disk offloading, (3) softmax approximation using semantically relevant token subsets, and (4) logits masking. Experiments on Llama-3.2 3B and Qwen-2.5 3B demonstrate up to 26x and 28x reduction in peak memory, enabling fine-tuning on resource-constrained devices.

**中文摘要：**
> 【领域】边缘设备微调 / 显存优化
> 【问题】在消费级硬件上使用 LoRA 微调 LLM 面临严苛的显存限制，峰值显存经常超出设备上限，特别对于十亿参数级模型和长上下文训练数据。
> 【方法】本文提出一套互补的显存压缩技术组合，在不牺牲模型质量的前提下降低显存占用：(1) 基模型量化 + 即时反量化；(2) 内存高效检查点（选择性激活缓存 + 磁盘卸载）；(3) 使用语义相关 Token 子集的 Softmax 近似；(4) Logits 掩码。
> **【效果】** 在 Llama-3.2 3B 和 Qwen-2.5 3B 上，峰值显存降低最高 **26×** 和 **28×**，使得资源受限设备上的微调成为可能。
