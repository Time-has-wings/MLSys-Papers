### 1. LiveServe: Interaction-Aware Serving for Real-Time Omni-Modal LLMs
**论文 ID：** [`2606.22983`](https://arxiv.org/abs/2606.22983)  
**作者：** Xiangyu Zhi, Peiqi Yin, Sheng Guan, Chenguang Zheng, James Cheng, Xiao Yan  
**分类：** cs.DC

**英文摘要：**
Realtime omni-modal LMs support speech-centric conversations where users stream inputs, hear generated audio, and interrupt freely. Existing Omni-LM serving systems still rely on throughput-oriented LLM scheduling and LRU KV offloading. These policies ignore audio playback and multi-turn reuse: they may generate tokens far beyond what users hear, wasting work after barge-in, and evict KV state needed in the next turn. LiveServe is an interaction-aware serving system for realtime Omni-LM interactions. It features a playback-aware scheduler that aligns generation with audio consumption, and a reuse-driven KV cache manager that keeps state across turns and suppresses offloading for near-future reuse. Deployed on a real GPU cluster, LiveServe reduces compute waste by up to 5.6×, KV offloading traffic by up to 14.3×, and TTFA (time to first audio) by up to 2.5× compared to state-of-the-art baselines.

**中文翻译：**
实时全模态语言模型支持以语音为中心的对话，用户可以流式输入、听到生成的音频并自由打断。现有的 Omni-LM 服务系统仍然依赖面向吞吐量的 LLM 调度和 LRU KV 卸载策略，忽略了音频播放和多轮复用需求。LiveServe 是一个面向实时 Omni-LM 交互的感知交互的服务系统，具有播放感知调度器和复用驱动的 KV 缓存管理器。在实际 GPU 集群上，LiveServe 相比现有方案将计算浪费减少最高 5.6 倍，KV 卸载流量减少最高 14.3 倍，首音频时间（TTFA）减少最高 2.5 倍。

### 2. FlexServe: A Fast and Secure LLM Serving System for Mobile Devices with Flexible Resource Isolation
**论文 ID：** [`2606.23370`](https://arxiv.org/abs/2606.23370)  
**作者：** Yinpeng Wu, Yitong Chen, Lixiang Wang, Jinyu Gu, Zhichao Hua, Yubin Xia  
**分类：** cs.CR, cs.LG, cs.OS

**英文摘要：**
Device-side Large Language Models (LLMs) have grown explosively, offering stronger privacy and higher availability than their cloud-side counterparts. During LLM inference, both the model weights and the user data are valuable, and attackers may compromise the OS kernel to steal them. ARM TrustZone is the de facto hardware-based isolation technology on mobile devices, used to protect sensitive applications from a compromised OS. However, protecting LLM inference with TrustZone incurs significant overhead due to resource contention between the normal and secure worlds. In this paper, we identify three key conflicts that cause this overhead: memory lock, CPU lock, and cache lock. To address these issues, we propose FlexServe, a system that enables flexible resource isolation for LLM serving on mobile devices. FlexServe introduces the concept of the shareable secure world and leverages both hardware and software co-design to resolve the three conflicts. We implement FlexServe on Kunpeng 920 and Rockchip 3588 platforms. Evaluation shows that FlexServe achieves up to 2.3× improvement in throughput and up to 46% reduction in latency compared to the baseline with TrustZone isolation.

**中文翻译：**
设备端大语言模型发展迅速，提供比云端更强的隐私性和可用性。然而在推理过程中，模型权重和用户数据都可能被攻击者通过入侵操作系统内核窃取。ARM TrustZone 是移动设备上事实标准的硬件隔离技术，但保护 LLM 推理会带来显著开销。FlexServe 提出了可共享安全世界的概念，通过硬件-软件协同设计解决内存锁、CPU 锁和缓存锁三个冲突问题。在 Kunpeng 920 和 Rockchip 3588 平台上，FlexServe 相比 TrustZone 基线实现最高 2.3 倍的吞吐量提升和 46% 的延迟降低。

## 🚀 推理加速

### 3. Concordia: JIT-Compiled Persistent-Kernel Checkpointing for Fault-Tolerant LLM Inference
**论文 ID：** [`2606.23521`](https://arxiv.org/abs/2606.23521)  
**作者：** Yuhang Gan, Yiwei Yang, Yuyi Li, Xiangyu Gao, Yichen Wang, Rain Jiang, Xiaoning Ding, Andi Quinn, Chen Tian  
**分类：** cs.DC, cs.LG

**英文摘要：**
Long-running LLM agents keep valuable state resident on GPUs: KV caches, request schedulers, communication state, and sometimes online adapters. Losing this state after a GPU or communicator failure can discard minutes to hours of work, yet existing recovery mechanisms either restart the whole serving stack or require application-specific checkpoint logic inside every attention and runtime component. This paper argues that fault tolerance for such workloads needs a GPU-resident execution context—a persistent-kernel runtime that can capture and restore GPU-side state without host involvement. We introduce Concordia, a JIT-compiled persistent-kernel checkpointing framework. Concordia compiles CUDA kernel code to a portable IR, transparently instruments it with checkpoint and restore primitives at compile time, and enables sub-millisecond GPU-side save/restore of KV caches and CUDA global state. On a cluster with NVIDIA H100 GPUs, Concordia checkpoints a 70B-parameter model's KV cache in 0.62 ms (512x faster than host-side techniques) and recovers an interrupted inference session in under 3 seconds with <5% throughput overhead during normal operation.

**中文翻译：**
长期运行的 LLM Agent 在 GPU 上保留了有价值的状态：KV 缓存、请求调度器、通信状态和在线适配器。GPU 或通信器故障后丢失这些状态会丢弃数分钟到数小时的工作。Concordia 是一个 JIT 编译的持久化内核检查点框架，将 CUDA 内核代码编译为可移植 IR，在编译时透明地注入检查点和恢复原语，实现亚毫秒级的 GPU 端 KV 缓存保存/恢复。在 NVIDIA H100 GPU 集群上，Concordia 在 0.62ms 内检查点 70B 模型的 KV 缓存（比主机端技术快 512 倍），在不到 3 秒内恢复中断的推理会话，正常运行时开销低于 5%。

## 💾 显存 / KV Cache 优化

### 4. FORGE: Fused On-Register Gradient Elimination for Memory-Efficient LLM Training
**论文 ID：** [`2606.22932`](https://arxiv.org/abs/2606.22932)  
**作者：** Dikshant Kukreja, Kritarth Prasad, Avinash Anand, Zhengkui Wang, Erik Cambria, Timothy Liu, Aik Beng Ng  
**分类：** cs.LG

**英文摘要：**
Reverse-mode differentiation computes every weight gradient, writes it to memory, and only then lets the optimizer read it back. This two-phase schedule sets the memory ceiling of modern training: at the seam between the phases, every layer's gradient is live at once. We argue that this materialized gradient is an artifact of how differentiation is staged, not a quantity that learning requires—and we eliminate it. FORGE folds the optimizer step into the backward pass and applies it one tile at a time, so gradients are consumed (and their memory freed) the moment they become available. The key insight is to fuse the backward pass of each operator with the optimizer's update rule, executed directly from on-chip registers without an intervening global-memory write. On a cluster of 8× NVIDIA H100 GPUs, FORGE reduces peak training memory by up to 38% for LLaMA-scale models and achieves up to 1.28× end-to-end throughput improvement over PyTorch FSDP, while preserving model quality.

**中文翻译：**
反向模式微分计算每个权重的梯度后写入内存，再让优化器读取——这种两阶段调度设定了现代训练的内存上限。FORGE 将优化器步骤折叠到反向传播中，每次只处理一个 tile，使得梯度在使用后立即释放其内存。核心创新是将每个算子的反向传播与优化器更新规则融合，直接在片内寄存器上执行，无需中间全局内存写入。在 8×NVIDIA H100 GPU 集群上，FORGE 将 LLaMA 规模模型的训练峰值内存降低最高 38%，端到端吞吐量相比 PyTorch FSDP 提升最高 1.28 倍。

### 5. Kamera: Unified Position-Invariant Multimodal KV Cache for Training-Free Reuse
**论文 ID：** [`2606.23581`](https://arxiv.org/abs/2606.23581)  
**作者：** Bole Ma, Jan Eitzinger, Harald Koestler, Gerhard Wellein  
**分类：** cs.DC, cs.AI, cs.CV

**英文摘要：**
Multimodal agents repeatedly re-examine the same video frames, UI screenshots, and rendered artifacts as their context window slides and reasoning iterates, yet every look-back re-encodes from scratch, because prefix caches serve reuse only at a fixed leading position. We show this recompute is avoidable, and identify exactly what naive KV reuse loses: the cross-chunk conditioning a chunk absorbs from its neighbours. This loss is asymmetric. The direct readout of a cached chunk is recovered exactly, while the cross-attention from other chunks to it is lost—an error that compounds under sliding-window and multi-turn reuse. We propose Kamera, a unified position-invariant KV cache that decouples cached state from absolute position. Kamera compresses each multimodal chunk into a position-invariant representation, tracks inter-chunk dependencies, and reconstructs the full KV cache on-demand with bounded error. On real multimodal agent traces, Kamera eliminates 92–99% of re-encoding FLOPs with negligible quality degradation (<2% accuracy drop), cutting end-to-end latency by 3.2–5.8×.

**中文翻译：**
多模态 Agent 在上下文窗口滑动和迭代推理时反复查看相同的视频帧、UI 截图和渲染产物，但每次回看都重新编码，因为前缀缓存仅在固定的前置位置支持复用。Kamera 提出统一的位置无关 KV 缓存，将缓存状态与绝对位置解耦。它将每个多模态块压缩为位置无关的表示，跟踪块间依赖关系，按需重建完整 KV 缓存并具有有界误差。在实际多模态 Agent 轨迹上，Kamera 消除了 92–99% 的重编码 FLOPs，质量下降可忽略（<2% 准确率下降），端到端延迟降低 3.2–5.8 倍。

### 6. HyperQuant: A Rate-Distortion-Optimal Quantization Pipeline for Large Language and Diffusion Models
**论文 ID：** [`2606.23406`](https://arxiv.org/abs/2606.23406)  
**作者：** Yuval Domb, Hadar Sackstein, Tomer Solberg  
**分类：** cs.LG, cs.AI

**英文摘要：**
We present HyperQuant (Hadamard, optimallY Packing, Entropy Rice-coding), a unified post-training quantization pipeline for the weights and the KV cache of large language and diffusion transformers. Across a suite of self-contained experiments (Table 1), HyperQuant outperforms the recent HIGGS scheme at every operating point from 3 to 5 bits per scalar (bps) on weights, and beats both TurboQuant and OCTOPUS on KV quantization down to 1.7 bps. Beyond the LLM setting, HyperQuant quantizes the 19B-parameter SDXL diffusion U-Net to 3 bps with negligible FID increase. The pipeline combines four stages: (1) a randomized Hadamard transform that mixes coefficient magnitudes for uniform quantization, (2) optimal bit-width packing via dynamic programming under a rate-distortion objective, (3) a sensitivity-aware grouping strategy that allocates bits to blocks based on their contribution to output error, and (4) entropy Rice-coding for lossless compression of the quantized representation.

**中文翻译：**
HyperQuant 是一个统一的后训练量化流水线，用于大语言模型和扩散模型的权重及 KV 缓存量化。它结合了四个阶段：(1) 随机 Hadamard 变换混合系数幅度以实现均匀量化；(2) 基于率失真目标的最优位宽打包；(3) 基于敏感度的分组策略；(4) 熵 Rice 编码实现无损压缩。在权重 3–5 bits 范围内全面优于 HIGGS 方案，KV 量化低至 1.7 bps 时优于 TurboQuant 和 OCTOPUS，并将 19B 参数的 SDXL U-Net 量化为 3 bps 仅带来可忽略的 FID 增加。

### 7. GRINQH: Graded Input-based Quantization Hierarchy for Efficient LLM Generation
**论文 ID：** [`2606.23419`](https://arxiv.org/abs/2606.23419)  
**作者：** Jette Oberländer, Jan Finkbeiner, Catherine M. Schöfmann, Emre Neftci  
**分类：** cs.LG, cs.AI

**英文摘要：**
Autoregressive decoding with LLMs is primarily bottlenecked by GPU memory bandwidth, especially in edge-computing settings. While quantization is essential for mitigating this bottleneck, most existing methods treat inference as a uniform process and fail to account for the asymmetry between the compute-bound prefill stage and the memory-bound decoding stage. We propose GRINQH (GRaded INput-based Quantization Hierarchy), a weight-only post-training quantization framework that accelerates decoding by adapting quantization precision based on input statistics. GRINQH assigns higher precision to weights processing outlier activations and lower precision to weights handling stable, low-variance activations, using only the input distribution as a guide. On LLaMA-2/3 and Mistral models at 7B–13B scale, GRINQH achieves 1.4–1.8× decoding speedup over FP16 with <0.5% accuracy degradation across standard benchmarks.

**中文翻译：**
LLM 的自回归解码主要受 GPU 内存带宽瓶颈限制，尤其在边缘计算场景中。GRINQH 提出基于输入统计信息自适应调整量化精度的权重后训练量化框架——为处理异常激活的权重分配更高精度，为处理稳定低方差激活的权重分配更低精度。在 LLaMA-2/3 和 Mistral 7B–13B 模型上，GRINQH 相比 FP16 实现 1.4–1.8 倍解码加速，标准基准测试准确率下降 <0.5%。

### 8. SVD-Surgeon: Optimal Singular-Value Surgery for Large Language Model Compression
**论文 ID：** [`2606.23568`](https://arxiv.org/abs/2606.23568)  
**作者：** Mahmoud Safari, Frank Hutter  
**分类：** cs.LG, cs.CL

**英文摘要：**
Large language models (LLMs) achieve remarkable performance across a wide range of tasks, but their deployment is constrained by substantial memory and compute requirements. Low-rank compression via singular value decomposition (SVD) is an effective remedy, but existing methods focus on how to factorize and which components to keep. We introduce SVD-Surgeon, a training-free method that brings the Optimal Brain Surgeon (OBS) framework to the singular-value basis. Treating each singular value as a parameter to be pruned, SVD-Surgeon computes the loss-saliency of each singular component using the empirical Fisher information matrix and selectively removes components with minimal impact on model output. Unlike prior SVD-based methods that use heuristic truncation, SVD-Surgeon provides a principled, saliency-driven selection. On LLaMA-2/3 and Mistral models, SVD-Surgeon achieves 30–50% compression with <1% accuracy loss, outperforming SVD-truncation by 3–5 perplexity points at the same compression ratio.

**中文翻译：**
大语言模型部署受限于大量内存和计算需求。SVD-Surgeon 将最优脑外科医生（OBS）框架引入奇异值基，将每个奇异值视为待剪枝的参数，使用经验 Fisher 信息矩阵计算每个奇异分量的损失显著性并选择性移除。在 LLaMA-2/3 和 Mistral 模型上，SVD-Surgeon 实现 30–50% 的压缩率，准确率损失 <1%，在相同压缩比下比 SVD 截断方法好 3–5 个困惑度点。

### 9. FlowTrain: Flow-Based Decoupled Training for Industrial-Grade Vision-Language Models
**论文 ID：** [`2606.23087`](https://arxiv.org/abs/2606.23087)  
**作者：** Zhida Jiang, Zhaolong Xing, Yang Pei, Xiaolong Chen, Yuanhang Xiao, Chengzhi Huang, Xiyu Liu, Haopeng Zhou, Runyu Ma, Jiaqin Guo, Xiaobo Zhou, Jiahao Liu, Jinkun Geng, Yufei Ren, Lianmin Zheng  
**分类：** cs.LG

**英文摘要：**
Industrial-grade distributed training of vision-language models (VLMs) remains far less efficient than that of unimodal LLMs. Existing solutions either follow a monolithic design that assigns uniform parallelism to heterogeneous modules or adopt a disaggregated deployment that separates modules while executing them as a batch-synchronized pipeline. In this paper, we highlight that the above solutions are still not sufficient, and VLM training can be further decoupled. To this end, we present FlowTrain, a decoupled training system for multimodal models. FlowTrain unifies a data-flow programming model with an asynchronous, pipeline-parallel execution engine that allows each module (vision encoder, connector, LLM) to train independently with heterogeneous parallelism configurations and micro-batch sizes. FlowTrain further introduces a disaggregated checkpointing mechanism and a dynamic pipeline rebalancing strategy. Evaluated on 128 NVIDIA A100 GPUs with a 13B-scale VLM, FlowTrain achieves 1.68× throughput improvement over state-of-the-art systems like DeepSpeed and Megatron, and reduces memory fragmentation by 40%.

**中文翻译：**
工业级视觉-语言模型的分布式训练效率远低于单模态 LLM。FlowTrain 是一个面向多模态模型的解耦训练系统，统一了数据流编程模型与异步流水线并行执行引擎，允许每个模块（视觉编码器、连接器、LLM）以异构并行配置和微批量大小独立训练。在 128 块 NVIDIA A100 GPU 上训练的 13B VLM 上，FlowTrain 相比 DeepSpeed 和 Megatron 等先进系统实现 1.68 倍吞吐量提升，内存碎片减少 40%。

### 10. The Energy Consumption of Transformer Fine-Tuning: A Roofline-Inspired Scaling Model
**论文 ID：** [`2606.23546`](https://arxiv.org/abs/2606.23546)  
**作者：** Mansour Zoubeirou a Mayaki  
**分类：** cs.LG, cs.AI, cs.AR, cs.CL, cs.DC

**英文摘要：**
Transformer-based models underpin modern natural language processing but incur rapidly growing computational and energy costs. As training scales in both model size and parallelism, accurately predicting energy consumption has become critical for sustainable and cost-aware system design. We present a framework for modeling the energy consumption of Transformer training on multiple GPUs. Using controlled architectural sweeps of BERT models, we relate measured energy to lightweight proxies for computation (FLOPs and memory access volume), deriving a practical energy predictor. Our model achieves a mean absolute percentage error of 22.8% across configurations spanning 2–64 GPUs. We use this predictor to analyze scaling trends and identify the most energy-efficient parallelism configurations under different throughput constraints, demonstrating how the framework can guide system design decisions for sustainable LLM training.

**中文翻译：**
基于 Transformer 的模型在带来巨大性能提升的同时也产生了快速增长的计算和能源成本。本文提出了一个在多个 GPU 上训练 Transformer 的能耗建模框架。通过 BERT 模型的控制架构扫描，将实测能耗与计算量（FLOPs 和内存访问量）的轻量代理指标关联，得出实用的能耗预测器，在 2–64 GPU 配置上达到 22.8% 的平均绝对百分比误差。该预测器可分析扩展趋势并识别不同吞吐量约束下最节能的并行配置，为可持续 LLM 训练的系统设计提供指导。
