### 1. You Only Index Once: Cross-Layer Sparse Attention with Shared Routing（CLSA）
- **ID:** 2606.06467v1
- **作者:** Yutao Sun, Yanqi Zhang, Li Dong, Jianyong Wang, Furu Wei
- **链接:** https://arxiv.org/abs/2606.06467

**英文摘要:** Long-context inference in modern LLMs is increasingly constrained by decoding efficiency, especially in reasoning-heavy settings where models generate long intermediate chains of thought. Existing sparse attention methods often face a practical efficiency-quality trade-off. Structured block sparse methods typically provide stronger acceleration but incur noticeable quality loss, while token sparse methods are usually more accurate yet deliver limited end-to-end speedup because top-k routing over the full cache remains expensive. In this work, we propose cross-layer sparse attention (CLSA), which is built on top of KV-sharing architectures such as YOCO. The core idea is to share not only the KV cache across cross-decoder layers, but also the routing index. A single indexer computes token-level top-k selection once and reuses the resulting index across layers, thereby preserving the fine-grained selectivity of token sparse attention while amortizing the routing overhead. The resulting architecture improves all major inference bottlenecks jointly, including pre-filling, KV-cache storage, and long-context decoding. Experiments across short-context and long-context benchmarks show that CLSA is both accurate and efficient, achieving up to 7.6x decoding speedup and 17.1x overall throughput improvement at 128K context.

**中文翻译:** 现代 LLM 的长上下文推理越来越受解码效率限制，尤其是在推理密集型场景中模型会产生长长的中间思考链。现有的稀疏注意力方法往往面临效率-质量的权衡：结构化块稀疏方法加速效果好但质量损失明显，而 token 稀疏方法更准确但端到端加速有限（因为对完整缓存进行 top-k 路由仍然昂贵）。本文提出跨层稀疏注意力（CLSA），建立在 YOCO 等 KV 共享架构之上。核心思想不仅是在跨解码器层之间共享 KV 缓存，还共享路由索引。单个索引器计算一次 token 级别的 top-k 选择，然后在各层复用结果索引，从而保留 token 稀疏注意力的细粒度选择性，同时摊销路由开销。实验表明，CLSA 在 128K 上下文中实现高达 7.6 倍解码加速和 17.1 倍整体吞吐提升。

### 2. Vortex: Efficient and Programmable Sparse Attention Serving for AI Agents
- **ID:** 2606.06453v1
- **作者:** Zhuoming Chen, Xinrui Zhong, Qilong Feng, Ranajoy Sadhukhan, Yang Zhou, Michael Qizhe Shieh, Zhihao Jia, Beidi Chen
- **链接:** https://arxiv.org/abs/2606.06453

**英文摘要:** Sparse attention is becoming increasingly important for serving large language models (LLMs) as generation lengths continue to grow. However, deploying and evaluating new sparse attention algorithms at scale remains highly engineering-intensive, slowing both human researchers and AI agents in exploring the sparse attention design. To address this challenge, we present Vortex, a system that combines a Python-embedded frontend language atop a page-centric tensor abstraction for expressing a broad range of sparse attention algorithms, with an efficient backend tightly integrated into modern LLM serving stacks. Vortex enables rapid prototyping, deployment, and evaluation of sparse attention algorithms, effectively translating their theoretical efficiency gains into real-world throughput improvements. As a result, Vortex substantially accelerates the design and iteration of sparse attention algorithms. First, AI agents use Vortex to automatically generate and refine diverse algorithms, the best reaching up to 3.46× higher throughput than full attention while preserving accuracy. Second, Vortex extends sparse attention to emerging architectures and very large models, reaching up to 4.7× higher throughput on the MLA-based GLM-4.7-Flash and 1.37× on the 229B-parameter MiniMax-M2.7 on NVIDIA B200 GPUs.

**中文翻译:** 随着生成长度的增长，稀疏注意力对 LLM 服务变得越来越重要。然而，大规模部署和评估新的稀疏注意力算法仍然是高度工程密集型的，减慢了研究人员和 AI 智能体探索稀疏注意力设计的进程。本文提出 Vortex，该系统结合 Python 嵌入的前端语言和以页面为中心的张量抽象来表达广泛的稀疏注意力算法，并配备紧密集成到现代 LLM 服务栈的高效后端。Vortex 使得稀疏注意力算法的快速原型设计、部署和评估成为可能，有效将其理论效率提升转化为实际的吞吐量改进。AI 智能体使用 Vortex 自动生成和优化多种算法，最佳方案在保持准确率的同时实现比全注意力高 3.46 倍的吞吐量。

### 3. RedKnot: Efficient Long-Context LLM Serving with Head-Aware KV Reuse and SegPagedAttention
- **ID:** 2606.06256v1
- **作者:** Yang Liu, ZhaoKai Luo, HuaYi Jin, ZhiYong Wang, RuoZhou He, BoYu Wang, Guanjie Chen, Junhao Hu
- **链接:** https://arxiv.org/abs/2606.06256

**英文摘要:** As the input length of large language model (LLM) serving continues to grow, the KV cache has become a dominant bottleneck in AI infrastructure. It limits GPU memory capacity, serving concurrency, cache reuse, and distributed scalability. We present RedKnot, a head-aware KV cache management system for LLM serving. RedKnot breaks the conventional monolithic KV cache abstraction by decomposing the KV cache along KV heads, whose importance and effective attention ranges vary significantly across serving scenarios. This head-level decomposition turns the KV cache from a monolithic tensor abstraction into a structured memory object, enabling RedKnot to uniformly support position-independent KV reuse, prefix KV compression, hot/cold KV separation, and distributed KV placement while preserving output fidelity and improving resource efficiency, without requiring model retraining or fine-tuning.

**中文翻译:** 随着 LLM 服务输入长度的持续增长，KV 缓存已成为 AI 基础设施中的主导瓶颈。它限制了 GPU 内存容量、服务并发性、缓存复用和分布式可扩展性。本文提出 RedKnot，一个面向 LLM 服务的头感知 KV 缓存管理系统。RedKnot 打破传统的单一 KV 缓存抽象，沿着 KV 头分解 KV 缓存——不同头的重视程度和有效注意力范围在不同服务场景中差异显著。这种头级别分解将 KV 缓存从单一的张量抽象转变为结构化内存对象，使 RedKnot 能够统一支持位置无关的 KV 复用、前缀 KV 压缩、热/冷 KV 分离和分布式 KV 放置，同时保持输出保真度和提高资源效率，无需模型重新训练或微调。

### 4. SlidingServe: SLO-Aware Sliding-Window Scheduling for LLM Inference
- **ID:** 2606.05933v1
- **作者:** Yuansheng Chen, Yue Zhang, Xuan Mo, Weigang Wu, Jialun Li
- **链接:** https://arxiv.org/abs/2606.05933

**英文摘要:** Existing LLM service systems rely on coarse-grained output constraints, making it difficult to effectively handle resource contention among multiple requests, resulting in low resource utilization efficiency and limited support for fine-grained quality of service (QoS) differentiation. We present SlidingServe, a sliding-window-driven SLO-Aware scheduling system for online LLM inference. SlidingServe designed a lightweight batch latency predictor to estimate the execution time of a batch. Based on this, SlidingServe uses SlidingChunker to combine information from the current iteration and the next iteration to achieve dynamic chunking and improve the overall system throughput while maintaining strict QoS guarantees. SlidingServe introduces Multi-Level Priority Sorter to sort candidate requests in order to balance fairness and efficiency. Additionally, when multiple requests within the same batch are at risk of SLO violating, SlidingServe introduces BatchConstructor, which uses dynamic programming to select the set of requests to execute in the current round, mitigating the SLO violation risk of critical requests. Our evaluation demonstrates that SlidingServe can improve service capacity by up to 30% compared to advanced scheduling systems under various load conditions, and further reduces the rate of SLO violation by 16%-53% under heavy-load inference mode.

**中文翻译:** 现有 LLM 服务系统依赖粗粒度的输出约束，难以有效处理多个请求之间的资源竞争，导致资源利用效率低下且对细粒度 QoS 区分的支持有限。本文提出 SlidingServe，一个滑动窗口驱动的 SLO 感知在线 LLM 推理调度系统。SlidingServe 设计了轻量级的批量延迟预测器来估计批次的执行时间，并使用 SlidingChunker 结合当前迭代和下一次迭代的信息来实现动态分块，在保持严格 QoS 保证的同时提高系统整体吞吐量。评估表明，与先进调度系统相比，SlidingServe 可将服务容量提升高达 30%，在高负载推理模式下将 SLO 违反率降低 16%-53%。

### 7. Mechanistic Insights into Functional Sparsity in Multimodal LLMs via CoRe Heads

- **链接:** https://arxiv.org/abs/2606.05843

**英文摘要:** While Multimodal Large Language Models (MLLMs) demonstrate remarkable proficiency on complex vision-language tasks, the mechanisms by which they extract query-relevant visual features from complex, noisy contexts remain opaque. In this paper, we present an in-depth interpretability study that uncovers a profound structural property within MLLMs: functional sparsity in cross-modal retrieval. ### 9. Tangram: Unlocking Non-Uniform KV Cache for Efficient Multi-turn LLM Serving
- **ID:** 2606.06302v1
- **作者:** Hyungmin Kim, Minsoo Kim, Hongseok Kim, Jungwook Choi
- **链接:** https://arxiv.org/abs/2606.06302
- **代码:** https://github.com/aiha-lab/TANGRAM

**英文摘要:** Multi-turn Large Language Model (LLM) serving is critical for consistent user experiences, yet the linear growth of the Key-Value (KV) cache imposes significant pressure on GPU memory and bandwidth. Non-uniform KV compression effectively preserves more information by considering the individual importance of each KV cache. However, such KV cache heterogeneity introduces various systemic challenges — including memory fragmentation, scheduling complexities, and diminished kernel utilization — which collectively lead to significant inefficiencies in existing LLM serving systems. To overcome these challenges, we present Tangram, a novel serving system designed to make Non-uniform KV caches practical. Tangram addresses systemic inefficiencies through three core techniques: (1) Deterministic Budget Allocation assigns a static memory footprint to each head based on its intrinsic pattern, entirely eliminating dynamic scheduling overhead and prefill stalls; (2) Head Group Page clusters attention heads with similar retention demands and manages them with independent, vectorized page tables, thereby maximizing physical memory reclamation; and (3) Ahead-of-Time (AOT) Load Balancing leverages static budget profiles to ensure uniform GPU utilization without runtime overhead. Experimental results show that Tangram improves throughput by up to 2.6x compared to existing baselines, while fully preserving model accuracy.

**中文翻译:** 多轮 LLM 服务对一致的用户体验至关重要，但 KV 缓存的线性增长对 GPU 内存和带宽施加了巨大压力。非均匀 KV 压缩通过考虑每个 KV 缓存的个体重要性来有效保留更多信息。然而，这种 KV 缓存异构性引入了各种系统性挑战——包括内存碎片化、调度复杂性和内核利用率下降——共同导致现有 LLM 服务系统的低效。本文提出 Tangram，一个使非均匀 KV 缓存变得实用的新型服务系统。通过三项核心技术：确定性预算分配、头分组页面管理和提前负载均衡，Tangram 相比现有基线将吞吐量提升高达 2.6 倍，同时完全保持模型精度。

### 12. SecRL-Prune: Structured Reinforcement Learning-Based Pruning of CodeLLMs
- **ID:** 2606.06254v1
- **作者:** Parsa Memarzadehsaghezi, Pooria Madani, Khalil El-Khatib
- **链接:** https://arxiv.org/abs/2606.06254

**英文摘要:** Large code language models (CodeLLMs) can generate and rewrite programs, enabling functionality-preserving code mutation that may be used to create diverse malware variants and evade signature-based detection. A key security question is whether this mutation capability survives model compression, which would make deployment feasible under limited hardware budgets. We evaluate SecRL-Prune on HumanEval using pass@k for execution correctness and var@k for code diversity across three 7B CodeLLMs at 10-30% compression. SecRL-Prune consistently preserves higher pass@k and var@k than recent structured pruning baselines under aggressive pruning.

**中文翻译:** 大型代码语言模型能够生成和重写程序，实现保持功能的代码变异。本文提出 SecRL-Prune，一个针对 CodeLLM 的结构化剪枝框架，它对前馈（MLP/FFN）通道进行操作。从预训练教师模型开始，使用强化学习（以师生 KL 散度作为奖励）学习逐层剪枝策略。为提高效率，缓存一次的教师模型 top-P 预测，然后将剪枝后的学生模型与这个紧凑目标进行比较，避免教师和学生同时在 GPU 内存中驻留。在 10-30% 压缩率下，SecRL-Prune 在三个 7B CodeLLM 上持续优于最新的结构化剪枝基线。

### 13. LLM-Based Porting of Optimized C++ to CUDA Through Deoptimization and Reoptimization
- **ID:** 2606.06063v1
- **作者:** Daichi Mukunoki, Ryo Mikasa, Shunichiro Hayashi, Tetsuya Hoshino, Takahiro Katagiri
- **链接:** https://arxiv.org/abs/2606.06063

**英文摘要:** When porting high-performance computing (HPC) code from CPU to GPU, CPU-oriented optimizations may obstruct LLM-based CUDA translation. We design and evaluate a Deopt-Reopt workflow that first simplifies the input C++ code and then retranslates and reoptimizes it for CUDA, comparing it against direct translation (Direct) on twelve HPC kernels with two LLMs in Single-shot and Iterative settings. In Single-shot, among 18 testable cases Deopt-Reopt was significantly faster among successful trials in five, but Direct was faster in three, so removing CPU-specific optimizations is not universally beneficial. In Iterative, repeated generation and repair narrow the mode gap. Deopt-Reopt is an effective but non-universal technique for LLM-based GPU porting, with gains that depend on the kernel, the model, the search budget, and the success rate.

**中文翻译:** 将高性能计算（HPC）代码从 CPU 移植到 GPU 时，面向 CPU 的优化可能会阻碍基于 LLM 的 CUDA 翻译。本文设计并评估了一个 Deopt-Reopt 工作流，首先简化输入的 C++ 代码，然后为 CUDA 重新翻译和重新优化，并与直接翻译进行比较。在 12 个 HPC 内核上的实验表明，Deopt-Reopt 是一种有效但非普适的基于 LLM 的 GPU 移植技术，其收益取决于内核类型、模型、搜索预算和成功率。

### 14. PC Layer: Polynomial Weight Preconditioning for Improving LLM Pre-Training
- **ID:** 2606.06470v1
- **作者:** Senmiao Wang, Tiantian Fang, Haoran Zhang, Yushun Zhang, Kunxiang Zhao, Alex Schwing, Ruoyu Sun
- **链接:** https://arxiv.org/abs/2606.06470
- **代码:** https://github.com/Empath-aln/PC-layer

**英文摘要:** We propose a preconditioning (PC) layer, a weight parameterization via polynomial preconditioner that ensures stable weight conditioning throughout LLM training. The PC module reshapes the singular-value spectrum of weight matrices via low-degree polynomial preconditioning. After training, the preconditioned weights can be merged back into the original architecture, incurring no inference overhead. We demonstrate the advantage of the proposed PC layer over standard transformers in Llama-1B pre-training, for both the AdamW and Muon optimizers. Theoretically, we justify this spectrum-control principle by proving that uniformly bounding each layer's singular values ensures geometric convergence of gradient descent to global minima, for certain deep linear networks.

**中文翻译:** 本文提出一种预条件（PC）层，通过多项式预条件器对权重进行参数化，确保整个 LLM 训练过程中权重的稳定条件。PC 模块通过低阶多项式预条件重塑权重矩阵的奇异值谱。训练完成后，预条件后的权重可以合并回原始架构，不产生任何推理开销。我们在 Llama-1B 预训练中展示了 PC 层相对于标准 Transformer 的优势，适用于 AdamW 和 Muon 两种优化器。理论上，我们通过证明均匀约束每层的奇异值可以确保梯度下降对某些深度线性网络的全局最小值的几何收敛，来证明这种频谱控制原理的合理性。
