### [An Interpretable Latency Model for Speculative Decoding in LLM Serving](https://arxiv.org/abs/2605.15051)
- **Authors**: Linghao Kong
- **arXiv**: https://arxiv.org/abs/2605.15051

**Abstract**:  
Speculative decoding (SD) accelerates large language model (LLM) inference by using a smaller draft model to propose multiple tokens that are verified by a larger target model in parallel. While prior work demonstrates substantial speedups in isolated or fixed-batch settings, the behavior of SD in production serving systems with dynamic batching remains poorly understood. We present an interpretable latency model for SD in serving contexts that decomposes end-to-end latency into draft, verification, and scheduling components. The model captures the interaction between draft acceptance rate, batch size dynamics, and memory bandwidth contention. We validate the model on vLLM and show that it predicts latency within 8% across diverse workloads. Using the model, we derive an adaptive SD policy that selects draft model size and speculation depth based on real-time load, improving throughput by up to 1.4x over static SD configurations.

### [Accelerating State-Vector Quantum Simulation on Integrated GPUs via Cache Locality Optimization](https://arxiv.org/abs/2605.15098)
- **Authors**: Gabriel Fernandes Thomaz
- **arXiv**: https://arxiv.org/abs/2605.15098

**Abstract**:  
The classical simulation of quantum algorithms is a crucial tool for circuit development, testing, and validation. Although acceleration using GPUs significantly reduces simulation time, most high-performance simulators rely on vendor-specific frameworks that target data-center hardware. To broaden accessibility, we investigate the performance of state-vector quantum simulation on integrated GPUs, which are ubiquitous in consumer and edge devices but possess limited memory bandwidth. We identify cache locality as the primary bottleneck and propose a locality-aware scheduling algorithm that reorders gate applications to improve temporal and spatial locality. Our implementation achieves up to 3.2x speedup over baseline schedules on AMD integrated GPUs and demonstrates competitive performance with discrete GPU simulators for small-to-medium circuit depths.

### [A Hardware-Aware, Per-Layer Methodology for Post-Training Quantization of Large Language Models](https://arxiv.org/abs/2605.14929)
- **Authors**: Earl Killian
- **arXiv**: https://arxiv.org/abs/2605.14929

**Abstract**:  
Scaled Outer Product (SOP) is a post-training quantization methodology for large language model weights, designed to deliver near-lossless fidelity at 4.5--6 bits per weight on hardware with per-layer LUT decode. The methodology combines per-layer search of fixed and dynamic codebook pairs selected from a hardware-aware palette, with a layer-specific sensitivity metric that guides bit allocation. We validate SOP on Llama-2, Llama-3, and Mistral families and show that it achieves perplexity within 1% of FP16 at 5 bits on average, while being deployable on commodity edge accelerators. The per-layer search completes in under 30 minutes on a single GPU, making it practical for production deployment pipelines.

### [Evolving Layer-Specific Scalar Functions for Hardware-Aware Transformer Adaptation](https://arxiv.org/abs/2605.14047)
- **Authors**: Kieran Carrigg
- **arXiv**: https://arxiv.org/abs/2605.14047

**Abstract**:  
Vision Transformers (ViTs) achieve state-of-the-art performance on challenging vision tasks, but their deployment on edge devices is severely hindered by the computational complexity and global reduction bottleneck imposed by layer normalization. Recent methods attempt to bypass this by replacing normalization with simpler affine transforms, but these are applied uniformly across layers. We propose to evolve layer-specific scalar functions using genetic programming, where each layer's normalization is replaced by a hardware-cost-aware expression tree. The search objective combines task accuracy with latency on the target device. On ImageNet-1k with MobileViT, our method reduces normalization latency by 68% with only 0.3% accuracy drop, and the evolved functions generalize to other vision tasks without re-search.

### [XFP: Quality-Targeted Adaptive Codebook Quantization with Sparse Outlier Separation for LLM Inference](https://arxiv.org/abs/2605.14844)
- **Authors**: N/A
- **arXiv**: https://arxiv.org/abs/2605.14844

**Abstract**:  
We introduce XFP, a dynamic weight quantizer for LLM inference that inverts the conventional workflow: the operator specifies reconstruction quality floors on per-channel cosine similarity (one strict, one lenient), and the quantizer automatically selects the bit width and codebook granularity needed to meet them. XFP separates sparse outliers into a narrow parallel stream, allowing the bulk of weights to be encoded with small, highly regular codebooks that map efficiently to SIMD units. On Llama-3-70B, XFP achieves 3.8 bits per weight on average while keeping 99.7% of channels within the strict floor, yielding 1.35x end-to-end throughput improvement over AWQ on NVIDIA H100. The outlier stream adds only 0.4% memory overhead and is fused into the dequantization kernel.

### [Widening the Gap: Exploiting LLM Quantization via Outlier Injection](https://arxiv.org/abs/2605.15152)
- **Authors**: N/A
- **arXiv**: https://arxiv.org/abs/2605.15152

**Abstract**:  
LLM quantization has become essential for memory-efficient deployment. Recent work has shown that quantization schemes can pose critical security risks: an adversary may release a model that appears benign in FP32 but degrades catastrophically when quantized to INT8 or INT4. We systematically study this attack surface and show that injecting targeted outlier activations during fine-tuning can widen the accuracy gap between FP and quantized inference by up to 47 percentage points on reasoning tasks, while leaving FP accuracy nearly untouched. We analyze the root cause (outlier amplification in low-bit GEMM) and propose a defense based on outlier-aware smoothing that restores quantized accuracy without retraining. Our findings highlight the need for quantization-aware model verification in security-critical deployments.

### [SemaTune: Semantic-Aware Online OS Tuning with Large Language Models](https://arxiv.org/abs/2605.15026)
- **Authors**: Georgios Liargkovas
- **arXiv**: https://arxiv.org/abs/2605.15026

**Abstract**:  
Online OS tuning can improve long-running services, but existing controllers are poorly matched to live hosts. They treat scheduler, power, memory, and I/O controls as black-box variables and optimize a scalar reward. This view ignores cross-knob policy structure, breaks down when application metrics drift, and cannot incorporate operator intent. We propose SemaTune, a semantic-aware tuning framework that uses an LLM to translate natural-language goals into structured reward programs and to synthesize exploration policies over semantically meaningful control groups. SemaTune reasons about OS-level semantics (e.g., 'reduce tail latency for the database without hurting throughput') and composes multi-objective rewards that are grounded in kernel-visible telemetry. We evaluate SemaTune on Linux hosts running production-style workloads and show that it converges to better configurations faster than Bayesian optimization and RL baselines, while remaining interpretable and robust to workload drift.

### [Time Domain Near Memory Computing Engine](https://arxiv.org/abs/2605.14162)
- **Authors**: Sarthak Antal
- **arXiv**: https://arxiv.org/abs/2605.14162

**Abstract**:  
The increasing computational demand of AI workloads has intensified the need for energy-efficient in-memory and near-memory computing architectures, particularly because data movement often consumes significantly more energy than computation itself. While fully digital architectures provide robust signal integrity, they suffer from area and energy overheads associated with ADCs and DACs. We propose a time-domain near-memory computing engine that encodes data into pulse widths and performs multiply-accumulate operations using time-digital converters. The design eliminates precision ADCs and achieves 15 TOPS/W at 8-bit precision in 22nm CMOS. We evaluate the engine on transformer attention layers and demonstrate 4.3x energy-delay product improvement over conventional digital near-memory architectures.

### [Beyond What to Select: A Plug-and-play Oscillatory Data-Volume Scheduling for Efficient Model Training](https://arxiv.org/abs/2605.14773)
- **Authors**: N/A
- **arXiv**: https://arxiv.org/abs/2605.14773

**Abstract**:  
Data selection accelerates training by identifying representative training data while preserving model performance. However, existing methods mainly focus on designing sample-importance criteria, i.e., deciding what to select, while typically fixing the selected data volume as the target ratio throughout training. Thus, they are often dynamic in sample identity but static in data volume. In this work, we revisit data selection from an optimization perspective and show that selected-data training induces an implicit regularization effect modulated by the instantaneous selection ratio. This reveals a key trade-off: lower ratios amplify selection-induced regularization, whereas higher ratios preserve data coverage and optimization fidelity. Motivated by this insight, we propose PODS, a Plug-and-play Oscillatory Data-volume Scheduling framework. Rather than introducing another sample-scoring metric, PODS serves as a lightweight module that dynamically schedules how much data to select over training. Under the target selection ratio, PODS alternates between low-ratio regularization phases and high-ratio recovery phases to exploit selection-induced regularization without sacrificing optimization stability. With its lightweight, ratio-level, and task-agnostic design, PODS is compatible with existing static and dynamic selection methods and broadly applicable across training paradigms. Experiments across various datasets, architectures, and tasks show that PODS consistently improves the efficiency-generalization trade-off, e.g., reducing ImageNet-1k training cost by 50% with improved accuracy and accelerating LLM instruction tuning by over 2x without performance degradation.
