### RouteBalance: Fused Model Routing and Load Balancing for Heterogeneous LLM Serving

**论文 ID：** [`2606.17949`](https://arxiv.org/abs/2606.17949) | **PDF：** [📄](https://arxiv.org/pdf/2606.17949)

**作者：** Wei Da, Evangelia Kalyvianaki

**类别：** cs.DC | **发布时间：** 2026-06-16

**Abstract (English)：**
Heterogeneous LLM serving stacks split scheduling into two layers that optimize in isolation: model routers pick a model from quality and cost signals while ignoring instance load, and serving load balancers optimize queues while ignoring quality. We present RouteBalance, a serving-aware scheduling layer that fuses both into a single online assignment over concrete model instances, jointly trading off quality, latency, and cost. A batched in-process predictor stack and dead-reckoned instance state keep the joint decision cheap on the request hot path (approx 32 ms at 12 req/s). On a 13-instance, 28-GPU heterogeneous cluster serving four model sizes, a single deployed RouteBalance stack traces the upper region of the three-way quality-cost-throughput frontier. Sweeping one weight vector reaches both the highest routing-decision quality (DeepEval 0.419, +0.013 over the strongest baseline, 95% CI [+0.005,+0.022]) and, at its cost-priority corner, per-request cost that ties the cheapest baseline. With router engineering equalized against concurrent-scoring baseline variants we build, its balanced preset serves at 2.8 s and 30 req/s, leading 2.6 to 4.1x ahead of enhanced BEST-Route at high load.

**摘要（中文）：**
RouteBalance 提出了一种面向异构 LLM 服务集群的融合调度层，将模型路由与负载均衡统一为单个在线分配决策。通过在请求热路径上使用批处理进程内预测器（~32ms @12 req/s），联合优化质量、延迟和成本。在 13 实例、28 GPU 的异构集群上，RouteBalance 实现了质量-成本-吞吐量前沿的最优区域。均衡模式下在 2.8s 延迟下达到 30 req/s 吞吐，在高负载下比增强版 BEST-Route 领先 2.6～4.1x。

### LUMEN: Coordinated Failure Recovery for Distributed LLM Serving

**论文 ID：** [`2606.17787`](https://arxiv.org/abs/2606.17787) | **PDF：** [📄](https://arxiv.org/pdf/2606.17787)

**作者：** Zhang Cao, Shujie Han, Juncheng Zhang, Yuanming Ren, Yongkun Li, Patrick P. C. Lee

**类别：** cs.DC | **发布时间：** 2026-06-16

**Abstract (English)：**
Modern large language model (LLM) serving clusters distribute inference requests across multiple worker processes on different GPUs, but failures are prevalent at scale. When a worker fails, the cluster simultaneously loses the failed worker's GPU-resident KV caches and serving capacity. Existing systems either restart interrupted requests from scratch or restore KV caches from fixed-neighbor checkpoints, both ignoring current cluster load. We present LUMEN, a fault-tolerant LLM serving system that treats recovery as a load-aware coordination problem across three decision points: checkpoint placement before failures, interrupted-request distribution at failure time, and serving capacity restoration during model reload.

**摘要（中文）：**
LUMEN 提出面向分布式 LLM 服务集群的容错系统。现有方法要么从头重启中断请求，要么从固定邻居恢复 KV cache 检查点，都不考虑当前集群负载。LUMEN 将恢复过程建模为负载感知的协同问题，涵盖三个决策点：故障前的 checkpoint 放置、故障时的请求分配、以及模型重载期间的服务容量恢复。通过原型实验和大规模仿真验证了显著的服务和恢复时间改进。

### Tropical: Enhancing SLO Attainment in Disaggregated LLM Serving via SLO-Aware Multiplexing

**论文 ID：** [`2606.16264`](https://arxiv.org/abs/2606.16264) | **PDF：** [📄](https://arxiv.org/pdf/2606.16264)

**作者：** Jinming Ma, Jiefei Chen, Xiuhong Li, Jiangfei Duan, Haojie Duanmu, Xingcheng Zhang, Chao Yang, Dahua Lin

**类别：** cs.DC | **发布时间：** 2026-06-15

**Abstract (English)：**
To guarantee service quality in LLM serving, it is essential to meet the latency constraints of both prefill (TTFT) and decode (TPOT). Non-disaggregated and disaggregated serving each have trade-offs in TTFT vs TPOT. Tropical introduces an SLO-aware multiplexing strategy that balances queuing time and interference, enabling both high TTFT and TPOT SLOs simultaneously. Evaluation shows it outperforms both state-of-the-art non-disaggregated and disaggregated systems, achieving up to 2.09 more requests within 90% SLO attainment.

**摘要（中文）：**
Tropical 提出了一种面向解耦 LLM 服务的 SLO 感知多路复用策略。非解耦和解耦架构在 TTFT 和 TPOT 上各有优劣：解耦架构中 prefill 对 decode 干扰小但排队时间长，非解耦架构排队时间短但 prefill 与 decode 相互干扰严重。Tropical 通过平衡排队时间与干扰，同时优化 TTFT 和 TPOT 的 SLO 达标率，在 90% SLO 下服务请求数最多提升 2.09 倍。

### The Price of Anarchy in Disaggregated Inference

**论文 ID：** [`2606.17081`](https://arxiv.org/abs/2606.17081) | **PDF：** [📄](https://arxiv.org/pdf/2606.17081)

**作者：** Athos Georgiou

**类别：** cs.AR, cs.AI, cs.DC, cs.GT, cs.PF | **发布时间：** 2026-06-11

**Abstract (English)：**
Disaggregated inference architectures physically separate prefill and decode phases onto distinct GPU pools. We provide the first formal game-theoretic analysis of this architecture, using NVIDIA Dynamo as a concrete case study. We model disaggregated serving as three coupled games: a two-player resource game, a selfish caching game over the hierarchical KV cache, and a congestion game for request routing. We characterize how GPU saturation induces regime transitions. Based on this analysis, we design an adaptive controller that detects saturation transitions in real time and adjusts routing parameters. On a 3-node NVIDIA B200 cluster running Dynamo with Nemotron-4-340B and Llama-3.1-70B, adaptive routing drops PoA-hat by 3.1x (66.4 to 21.5) in the saturated phase at a 13% throughput cost.

**摘要（中文）：**
本文首次对解耦推理架构进行了正式的博弈论分析，以 NVIDIA Dynamo 为具体案例。将解耦服务建模为三个耦合博弈：prefill 与 decode 池间的两玩家资源博弈、分层 KV cache 上的自私缓存博弈、以及请求路由拥塞博弈。揭示了 GPU 饱和如何引发博弈支付结构的相变，并设计了自适应控制器实时检测饱和并调整路由参数。在 3 节点 B200 集群上，70B 模型在饱和阶段 PoA-hat 降低 3.1x（66.4→21.5），仅牺牲 13% 吞吐。

### Latency Prediction for LLM Inference on NPU Systems

**论文 ID：** [`2606.18042`](https://arxiv.org/abs/2606.18042) | **PDF：** [📄](https://arxiv.org/pdf/2606.18042)

**作者：** Juhyun Park, Seungwoo Jeong, Jingyu Lee, Kyungyong Lee

**类别：** cs.DC | **发布时间：** 2026-06-16

**Abstract (English)：**
Deploying LLMs requires exploring a large configuration space spanning parallelization strategies, batching techniques, and scheduling policies. We present LENS, a latency estimator that predicts NPU inference latency without knowledge of the microarchitecture or compiler. LENS profiles each bucket with two end-to-end measurements and composes results to predict latency for arbitrary input-output length combinations. Validated across NPUs from multiple vendors and several LLMs, achieving a mean prediction error of 2.15%.

**摘要（中文）：**
LENS 是一个面向 NPU（神经网络处理单元）的 LLM 推理延迟预估器。不需要了解底层微架构或编译器细节，通过在每个 bucket 上做两次端到端测量来 profile，然后组合预测任意输入-输出长度组合的延迟。在多厂商 NPU 和多个 LLM 上验证，平均预测误差仅 2.15%。

### SMEPilot: Characterizing and Optimizing LLM Inference with Scalable Matrix Extensions

**论文 ID：** [`2606.16332`](https://arxiv.org/abs/2606.16332) | **PDF：** [📄](https://arxiv.org/pdf/2606.16332)

**作者：** Feiyang Chen, Haibo Chen

**类别：** cs.DC, cs.AI, cs.PF | **发布时间：** 2026-06-15

**Abstract (English)：**
Modern CPUs increasingly integrate matrix extensions (e.g., Arm SME) that provide high-throughput matrix execution. We present SMEPilot, an LLM inference engine that selects CPU-only, SME-only, or cooperative SME+CPU execution for each operator shape. SMEPilot partitions matrix work across SME and CPU cores at tile granularity, overlaps SME-suitable matrix stages with CPU-suitable vector stages in attention, and maintains layout state for tensor reuse. Across Llama-3.2-3B, Qwen3-4B, and Qwen3-30BA3B on phone, PC, and server platforms, SMEPilot improves end-to-end inference performance by up to 3.94x.

**摘要（中文）：**
SMEPilot 是一个利用 CPU 矩阵扩展指令（Arm SME）加速 LLM 推理的引擎。它根据每个算子形状动态选择纯 CPU、纯 SME、或 SME+CPU 协同执行模式。以 tile 粒度在 SME 和 CPU 核心间划分矩阵计算，在 attention 中将适合 SME 的矩阵阶段与适合 CPU 的向量阶段重叠。在手机、PC 和服务器平台上对 Qwen3、Llama 等模型实现最高 3.94x 端到端推理加速。

### Prefill/Decode-Aware Evaluation of LLM Inference on Emerging AI Accelerators

**类别：** cs.AR, cs.AI, cs.DC | **发布时间：** 2026-06-14

**Abstract (English)：**
This paper presents a phase-aware evaluation of LLM inference performance across GPUs and emerging AI accelerators using Llama2-7B. By separately measuring Prefill and Decode performance, we reveal that accelerator advantages differ by phase and metric. ### AnchorKV: Safety-Aware KV Cache Compression via Soft Penalty with a Refusal Anchor

**论文 ID：** [`2606.17872`](https://arxiv.org/abs/2606.17872) | **PDF：** [📄](https://arxiv.org/pdf/2606.17872)

**作者：** Ning Ni, Yingjie Lao

**类别：** cs.LG, cs.AI | **发布时间：** 2026-06-16

**Abstract (English)：**
The KV cache becomes a dominant LLM inference bottleneck. Recent compression methods retain only a subset of attention-relevant tokens, but their compression policies either fail to defend against jailbreak attacks or degrade safety alignment under aggressive eviction. We propose AnchorKV, a drop-in modification to KV cache compression that biases token retention scores away from directions in key space associated with harmful prompts. AnchorKV constructs an offline safety anchor by adapting a difference-of-means representation engineering approach to the layer-specific key projection space used in KV caching. A soft penalty token selection rule trades a small amount of utility for substantially improved safety alignment.

**摘要（中文）：**
AnchorKV 提出了面向 KV cache 压缩的安全感知方法。现有压缩方法在激进驱逐策略下无法防御 jailbreak 攻击或损害安全对齐。AnchorKV 构建离线安全锚点，通过差异均值表征工程技术将 token 保留分数偏向远离有害 prompt 的方向，以极小的效用损失换取显著的安全对齐改善。可直接替换现有压缩器的 drop-in 方案。

### CacheWise: Understanding Workloads and Optimizing KVCache Management for Efficiently Serving LLM Coding Agents

**论文 ID：** [`2606.16824`](https://arxiv.org/abs/2606.16824) | **PDF：** [📄](https://arxiv.org/pdf/2606.16824)

**作者：** Shubham Tiwari, Tapan Chugh, Nash Rickert, Simon Peter, Ratul Mahajan, Haiying Shen

**类别：** cs.DC, cs.OS | **发布时间：** 2026-06-15

**Abstract (English)：**
Coding agents are a fast-growing LLM application, executing as long-running closed-loop sessions. We collect real-world coding assistant traces and analyze their serving behavior. Our analysis shows that coding agent sessions repeatedly reuse large prefixes and create sustained KVCache pressure that conventional policies handle poorly. We present CacheWise, a KVCache management layer combining prefix-aware scheduling with reuse-aware eviction guided by lightweight predictions from tool call metadata. Implemented in vLLM, CacheWise reduces KVCache evictions by up to 2-2.6x and improves total agent session completion time by up to 3.5x.

**摘要（中文）：**
CacheWise 专注于 LLM coding agent 场景的 KVCache 管理。通过分析真实 coding assistant 轨迹，发现 agent 会话反复重用长前缀，对传统策略造成持续压力。CacheWise 结合前缀感知调度和重用感知驱逐策略，利用工具调用元数据的轻量预测指导缓存管理。基于 vLLM 实现，减少 KVCache 驱逐 2～2.6x，agent 会话完成时间提升最高 3.5x。

### SwiftCache: Efficient LLM Serving for Multi-turn Conversations with Heterogeneous KV Cache Sharing

**论文 ID：** [`2606.16135`](https://arxiv.org/abs/2606.16135) | **PDF：** [📄](https://arxiv.org/pdf/2606.16135)

**作者：** Jianmin Hu, Minxian Xu, Sa Wang, Chong Ma, Min Shen, Kejiang Ye, Lin Qu, Chengzhong Xu

**类别：** cs.DC | **发布时间：** 2026-06-15

**Abstract (English)：**
Multi-turn conversations in LLM applications cause historical tokens to accumulate, forcing KV cache offload to CPU/SSD and limiting context length. SwiftCache enables heterogeneous models to share underutilized GPU memory and NVLink bandwidth within a server. Models with low KV cache demand donate idle GPU memory to store prefix caches of high-demand models, avoiding slow PCIe transfers. SwiftCache keeps only the KV cache of the currently active layer in local GPU memory. Experiments show SwiftCache reduces P99 TTFT by up to 69% and extends maximum context length by up to 3.98x compared to vLLM and SGLang.

**摘要（中文）：**
SwiftCache 提出异构模型间 KV cache 共享方案。在多轮对话场景中，历史 token 累积导致 KV cache 被迫卸载到 CPU/SSD。SwiftCache 让低 KV cache 需求的模型捐赠空闲 GPU 内存给高需求模型存储 prefix cache，通过 NVLink 传输避免低速 PCIe。同时仅保留当前活跃层的 KV cache 在本地 GPU 内存中。相比 vLLM 和 SGLang，P99 TTFT 降低最高 69%，最大上下文长度扩展最高 3.98x。

### Models Take Notes at Prefill: KV Cache Can Be Editable and Composable

**论文 ID：** [`2606.17107`](https://arxiv.org/abs/2606.17107) | **PDF：** [📄](https://arxiv.org/pdf/2606.17107)

**作者：** Bojie Li

**类别：** cs.LG, cs.AI | **发布时间：** 2026-06-14

**Abstract (English)：**
Prefix caching reuses prefill only across an exactly shared prefix. This paper shows two capabilities: (1) KV cache is editable - editing the field's own key/value recovers the decision with ~1% compute. (2) KV cache is composable - a precompiled skill can be RoPE-repositioned and spliced into any context, indistinguishable from full recompute at O(L) rather than O(L^2) TTFT. A unified edit+compose agent stays decision-identical to recompute at up to 14.9x lower latency. In an online vLLM benchmark it keeps the prefix cache-aligned (98.5% hit-rate), cutting p90 TTFT by 53-398x.

**摘要（中文）：**
本文揭示了一个重要发现：KV cache 是可编辑和可组合的。Prefill 阶段模型已将场条件结论写入下游 notes，因此（1）编辑特定字段的 key/value 向量即可纠正决策，仅需约 1% 计算量；（2）预编译的 skill 可通过 RoPE 重定位拼接到任意上下文中，O(L) TTFT 且与完全重计算不可区分。统一编辑+组合 agent 延迟最高降低 14.9x；在线 vLLM benchmark 中保持 98.5% 前缀缓存命中率，p90 TTFT 降低 53～398x。

### ITME: Inference Tiered Memory Expansion with Disaggregated CXL-Hybrid Memories

**作者：** Hakbeom Jang, Younghoon Min, Sunwoong Kim, Taeyoung Ahn, Hanyee Kim, Youngpyo Joo, Hoshik Kim, Jongryool Kim

**类别：** cs.DC | **发布时间：** 2026-06-10

**Abstract (English)：**
The rapid shift toward agentic and long-context workloads in LLMs pushes beyond individual server capacity toward disaggregated shared storage. We propose ITME, which leverages CXL-hybrid memory to present TB-scale byte-addressable remote memory expansion. Key insight: deterministic access patterns of model weights and prefix caches enable proactive data movement across the memory-storage hierarchy. **摘要（中文）：**
ITME 利用 CXL 混合内存提供 TB 级字节寻址远程内存扩展，解决长上下文和 agent 工作负载下 LLM 推理的显存瓶颈。关键洞察：模型权重和前缀缓存的确定性访问模式允许系统主动管理内存-存储层级间的数据移动。使用 SK Hynix CMM 和 PCIe Gen5 NVMe SSD 验证，配合 FPGA 原型。通过扩展 KV cache 容量超越本地主机内存限制，实现最高 35.7% 吞吐提升。

### AIMER: Calibration-Free Task-Agnostic MoE Expert Pruning

**Abstract (English)：**
Mixture-of-Experts (MoE) language models increase parameter capacity without proportional per-token computation, yet deployment requires storing the full expert pool. ### DICE: Diffusion Large Language Models Excel at Generating CUDA Kernels
